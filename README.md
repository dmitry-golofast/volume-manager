# Volume Manager

Форма создания нового Volume для админ-панели системы управления хранением данных.

## Стек

- Vue 3
- TypeScript
- Composition API и `<script setup>`
- Vite
- PrimeVue
- Tailwind CSS
- Vee-Validate
- Zod
- Vue I18n
- Vitest
- ESLint

## Что делает форма

Пользователь заполняет параметры нового тома:

- `name` — имя тома
- `type` — тип тома: `block` или `file`
- `size` — числовой размер
- `unit` — единица измерения

В шапке есть переключатель системы измерения:

- десятичная: `KB`, `MB`, `GB`
- двоичная: `KiB`, `MiB`, `GiB`

Выбранная система влияет на список доступных единиц измерения. При отправке размер конвертируется в байты и рядом с формой показывается объект, который должен уйти на бэкенд:

```json
{
  "name": "fast-storage",
  "type": "block",
  "sizeBytes": 100000000000
}
```

## Валидация

Валидация формы сделана через Vee-Validate и Zod.

Правила:

- имя обязательно
- имя должно состоять только из латиницы, цифр и дефиса
- размер обязателен
- размер должен быть числом больше нуля
- тип тома обязателен
- единица измерения обязательна

Проверка уникальности имени выполняется через локальный mock API из `src/mock.js`:

```ts
mockApi.checkName(name)
```

Метод имитирует задержку и случайные ошибки, HTTP-сервер для него не нужен.

## UI

Интерфейс собран на PrimeVue, layout и визуальная доводка сделаны через Tailwind CSS. Тексты интерфейса подключены через Vue I18n, текущая локаль — русский язык.

При загрузке страницы карточки формы появляются с анимацией. Для пользователей с `prefers-reduced-motion: reduce` анимация отключается.

## Установка

```bash
npm install
```

## Запуск в режиме разработки

```bash
npm run dev
```

После запуска приложение будет доступно по адресу, который выведет Vite, обычно:

```text
http://localhost:5173/
```

## Проверки

Запустить ESLint:

```bash
npm run lint
```

Запустить тесты:

```bash
npm test
```

Собрать production build:

```bash
npm run build
```

Локальный preview production-сборки:

```bash
npm run preview
```
